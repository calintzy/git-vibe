## 2. 기능 설계

### 2.1 S-01: `--share` SVG 이미지 생성 (HIGH)

**목표**: 바이브 체크 결과를 SNS 공유 가능한 이미지로 출력

**CLI 인터페이스**:
```bash
git-vibe --share                    # vibe-report.svg 생성
git-vibe --share --format png       # vibe-report.png 생성 (resvg 필요)
git-vibe --share -o my-vibe.svg     # 파일명 지정
```

**출력 이미지 구성**:
```
┌─────────────────────────────────────────────┐
│  🎭 Vibe Check: my-project                  │
│  ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━   │
│                                              │
│  Overall Vibe: 😎 Chill (74/100)             │
│                                              │
│  📊 Scores                                   │
│  Commit Frequency  ████████████░░  80        │
│  Hotspot Health    ██████████░░░░  72        │
│  Ghost Files       ████████████░░  85        │
│  Commit Mood       █████████░░░░░  65        │
│  Bus Factor        ██████░░░░░░░░  50        │
│  Activity Trend    ██████████████  95        │
│                                              │
│  🏋️ Bus Factor: 3 👍                         │
│  🎭 Mood: 😊 45% 😰 32% 🧹 18% 💀 5%       │
│                                              │
│  ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━   │
│  Generated by git-vibe · github.com/...      │
└─────────────────────────────────────────────┘
```

**기술 선택**:

| 옵션 | 장점 | 단점 | 선택 |
|------|------|------|:----:|
| SVG 직접 생성 (문자열) | 의존성 0, 빌드 빠름 | 복잡한 레이아웃 어려움 | ✅ |
| resvg (SVG→PNG) | 고품질 PNG | 큰 의존성, 빌드 느림 | feature flag |
| plotters | 차트 라이브러리 | 과도한 의존성 | ❌ |

**결정**: SVG를 문자열 템플릿으로 직접 생성 (의존성 0). PNG 변환은 `resvg` feature flag로 선택적 지원.

**새 파일**: `src/output/svg.rs`

### 2.3 S-03: GitHub Action (MEDIUM)

**목표**: PR에 바이브 체크 결과를 자동으로 코멘트

**Action 구조**:
```
.github/
  actions/
    git-vibe/
      action.yml       # Action 메타데이터
      Dockerfile        # Rust 빌드 환경
```

**action.yml**:
```yaml
name: 'Git Vibe Check'
description: 'Analyze your codebase vibe and post results'
inputs:
  period:
    description: 'Analysis period (e.g., 3m, 6m, 1y)'
    default: '6m'
  comment:
    description: 'Post result as PR comment'
    default: 'true'
runs:
  using: 'docker'
  image: 'Dockerfile'
```

**PR 코멘트 형식**:
```markdown
## 🎭 Vibe Check

**Overall Vibe: 😎 Chill (74/100)**

| Metric | Score |
|--------|-------|
| Commit Frequency | 80 |
| Hotspot Health | 72 |
| ...
```

**구현 방식**: `git-vibe --json` 출력을 파싱하여 Markdown 코멘트 생성 (셸 스크립트 기반)

## 3. 기술 결정 사항

### 3.1 확정 사항

| 결정 | 선택 | 이유 |
|------|------|------|
| SVG 생성 방식 | 문자열 템플릿 | 외부 의존성 0, 빌드 빠름, Rust 문자열로 충분 |
| PNG 지원 | `resvg` feature flag | 기본 빌드에 영향 없이 선택적 지원 |
| 배지 스타일 | shields.io 호환 | 개발자에게 익숙한 형태 |
| GitHub Action | Docker 기반 | 격리된 빌드 환경, cross-platform |
| 이미지 크기 | 1200x630px (OG 이미지) | Twitter/Facebook Open Graph 표준 |

### 3.2 Cargo.toml 변경

```toml
[features]
default = []
png = ["resvg"]

[dependencies]
# 기존 의존성 유지
resvg = { version = "0.44", optional = true }
```

### 3.3 CLI 변경

```rust
pub struct Cli {
    // ... 기존 ...

    /// 공유용 이미지 생성
    #[arg(long = "share")]
    pub share: bool,

    /// 이미지 출력 형식 (svg, png)
    #[arg(long = "format", default_value = "svg")]
    pub format: String,

    /// 출력 파일 경로
    #[arg(short = 'o', long = "output")]
    pub output: Option<PathBuf>,

    /// README 배지 생성
    #[arg(long = "badge")]
    pub badge: bool,
}
```
