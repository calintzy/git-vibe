# git-vibe — 코드베이스 바이브 체크 도구

> **주의**: "git-vibe" 이름이 GitHub에서 이미 사용 중. 최종 이름 후보: `repo-vibe`, `code-aura`, `vibecheck`, `git-mood`

## 1. 프로젝트 개요

| 항목 | 내용 |
|------|------|
| **이름** | git-vibe (가칭, 변경 필요) |
| **한줄 소개** | Git 히스토리를 분석해 이모지 기반 "바이브 체크" 리포트를 생성하는 재미있는 CLI |
| **카테고리** | Developer Tooling / Fun & Viral |
| **라이선스** | MIT |
| **난이도** | 중 |
| **예상 개발 기간** | MVP 2~4주 |

## 2. 문제 정의

### 현재 상황
- 새 프로젝트에 합류하면 코드베이스 상태를 파악하기 어려움
- 오픈소스 라이브러리 선택 시 "코드 품질"을 직관적으로 판단할 수 없음
- 기존 도구(gitinspector, git-stats)는 딱딱한 통계만 제공 → 재미없고 공유하고 싶지 않음

### 핵심 고통점
1. **직관적 파악 불가**: "이 프로젝트 괜찮아?"에 대한 빠른 답이 없음
2. **통계의 지루함**: 숫자 나열은 감정적으로 와닿지 않음
3. **공유 불가**: 기존 도구 결과를 SNS에 올리고 싶지 않음
4. **숨겨진 리스크**: 핫스팟, 유령 파일, 기술 부채가 보이지 않음

### 바이브 등급 시스템

| 점수 | 바이브 | 이모지 | 설명 |
|------|-------|-------|------|
| 90~100 | Zen | 🧘 | 완벽하게 관리되는 프로젝트 |
| 80~89 | Chill | 😎 | 건강하고 여유로운 상태 |
| 70~79 | Active | 💪 | 활발하지만 관리 필요 |
| 60~69 | Tense | 😬 | 기술 부채 쌓이는 중 |
| 50~59 | Stressed | 😰 | 핫스팟 많고 리스크 높음 |
| 40~49 | Chaotic | 🔥 | 불타는 중 |
| 0~39 | Abandoned | 💀 | 방치된 프로젝트 |

### 작동 방식
```bash
git-vibe                      # 현재 레포 바이브 체크
git-vibe --period 6m          # 최근 6개월 기준
git-vibe --compare main dev   # 브랜치 비교
git-vibe --share              # 공유 가능한 이미지/URL 생성
git-vibe --json               # JSON 출력 (CI 통합)
git-vibe --leaderboard        # 팀원 기여 순위
```

## 6. 기술 스택

| 구성 요소 | 기술 | 선택 이유 |
|----------|------|----------|
| **언어** | Rust | 빠른 git 히스토리 분석, 단일 바이너리 |
| **Git 파싱** | gix (gitoxide) | Rust 네이티브 Git 구현, 매우 빠름 |
| **CLI 프레임워크** | clap | Rust CLI 표준 |
| **터미널 출력** | ratatui (선택) / comfy-table + colored | 이모지 + 색상 출력 |
| **이미지 생성** | resvg (SVG→PNG) | 공유용 이미지 렌더링 |
| **감정 분석** | 키워드 매칭 (커스텀) | LLM 불필요, 빠르고 오프라인 |

### git-vibe의 차별점
1. **재미**: 이모지, 바이브 등급 → 공유하고 싶어지는 결과물
2. **통합 분석**: 핫스팟 + 유령 + 감정 + Bus Factor를 한번에
3. **바이럴 설계**: 공유 이미지, 배지, 웹 뷰어 기본 제공
4. **Actionable**: 단순 통계가 아닌 "무엇을 해야 하는지" 제시

## 11. 바이럴 전략 (핵심)

이 프로젝트의 성패는 **바이럴**에 달려 있음.

### 런칭 전
1. 유명 레포 10개 바이브 체크 결과 미리 준비 (React, Vue, Linux, etc.)
2. 결과 이미지를 SNS용으로 최적화 (Open Graph 사이즈)
3. "가장 Zen한 레포 vs 가장 Chaotic한 레포" 비교 콘텐츠

### 런칭
4. **Hacker News**: "Show HN: Your codebase has a vibe — find out what it is"
5. **Reddit**: `r/programming`, `r/webdev`에 유명 레포 분석 결과와 함께 게시
6. **X/Twitter**: 유명 레포 바이브 결과 쓰레드

### 런칭 후
7. 사용자가 자기 레포 결과를 공유하도록 유도 (`--share`가 핵심)
8. "Vibe Leaderboard" — 가장 Zen한 오픈소스 프로젝트 랭킹 사이트
9. README 배지 문화 확산 (`![Vibe](https://vibe.dev/badge/user/repo)`)
